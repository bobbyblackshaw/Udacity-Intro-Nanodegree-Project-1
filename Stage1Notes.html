Stage 0 Notes

The Internet

The internet is comprised of servers all over the world which are accessed by users of computers. The computers use internet browsers to find and view the files which are in the servers. In order to do this the computer and servers use HTTP, Hypertext Transfer Protocol. This ensures that the information is in a consistent format for easy transfer and understanding.

The Language of the Internet

The basic file type that the internet is comprised of is HTML, or Hypertext Markup Language. This is the language and filetype that web pages are made of. Being a markup language, it uses tags. Tags surround the content you want to implement into your webpage, and each tag has a unique characteristic. You must nest your tags for them to work as you want. Not all tags must have an end tag though. Tags that use no content do not need an end tag, and are called void tags.

Different Kinds of Elements

So far, we have learned of two kinds of elements: Inline and Block elements. Block elements put everything in them into an invisible box. This box can have added features, like controlling the dimensions of the space it takes up. Inline elements on the other hand are independent fo the content around them.

Stage 1 Notes

Development Tools

In stage 1, we used Google Chrome's development tools to look at the source code of any website. These tools are very useful because they allow you to quickly access information about how a website is structered and styled. By right clicking an element on the website and "inspecting" it, the development tools will go directly to the corresponding HTML. There, you can make changes or simply examine the details of the element and everything that it is comprised of.

Everything Is Boxes

Another big part of this stage is thinking about a web page as a bunch of boxes, usually with boxes inside of the bigger boxes. Every element is a box, even if it looks like a circle. This is so that it is easier to organize a web page and easier to break down a page into its basic elements visually. The tag we use to make these boxes is the div tag, which is a Block type element.

Styles and CSS

When there is an important element, it is usually accompanied by an attribute. One example we have been learning is to assign a class attribute, basically a label, so that we can easily reference everything of that class later on. We will be using CSS to style our web page, and can assign a style to a class. This makes styling very easy to do, and less tedious.

Box Sizing and Positioning

Every element in HTML is a box. One major function of using these boxes is being able to control their size and position.

Sizing

When sizing, it can be tough to get it just how you want it. Some techniques for this are setting the sizes in percentages instead of in pixels, and to use the box-sizing attribute and set it to border-box. Another important variable to keep in mind is that different browsers can display the same code differently. In order to keep a consistent look throughout all browsers, you must use special prefixes with the attributes. The prefixes are unique for each browser. A couple examples are web-kit for safari, and moz for mozilla firefox.

Positioning

Div elements are block elements, so they take up the entire width of the page. A cool trick to help with this potential problem is the rule display: flex;. This overrides the div element wanting to take the entire width of a page and lets divs appear next to each other.
